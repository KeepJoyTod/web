# 全局规则 - 减少 AI 幻觉

本文件用于约束 AI 在本地项目中的回答和改动方式。目标不是让回答变慢，而是让结论可追溯、可验证、可复现。

## 先验证再回答

- 回答代码、配置、脚本、数据库、接口相关问题前，必须先搜索或读取实际文件，不能凭记忆、经验或项目名猜测。
- 优先使用当前环境可用的搜索和读取工具，例如 `rg`、`grep`、`Get-ChildItem`、`Select-String`、`find`、`glob`、`read`、`cat`、`Get-Content` 等；工具名不一致时，使用等价工具完成验证。
- 提到文件路径前，必须验证文件或目录确实存在。
- 提到函数、类、接口、字段、SQL 表、配置项前，必须搜索确认其存在、位置和当前签名或结构。
- 涉及依赖版本、框架能力、命令参数、外部 API、法律规则、价格、日期、状态等可能变化的信息时，必须查官方文档、源码、锁文件、配置文件或可信来源。

## 明确区分事实和推断

- 回答时优先给出已验证事实，再给结论。
- 如果根据代码做推断，必须明确写出“根据以上代码推断”或同等表达。
- 如果信息无法验证，必须明确说明“这个信息我无法验证”，并给出可行的验证办法。
- 不要用看似确定的语气填补未知信息。

## 禁止编造

- 禁止编造不存在的文件、目录、URL、API、函数、类、库、命令、配置项、版本号、表结构或业务规则。
- 禁止把常见框架默认行为当成当前项目事实，除非已在当前项目或官方文档中确认。
- 禁止在没有读取错误日志、测试输出或相关代码的情况下断言根因。
- 不确定时必须说“不确定”或“需要验证”，而不是给出猜测型结论。

## 引用来源

- 回答涉及本地代码或配置时，必须引用具体文件路径和行号。
- 引用格式使用 `file_path:line_number`，例如 `back/src/main/java/com/example/App.java:12`。
- 引用外部资源时必须注明来源；外部 URL 或可能变化的链接必须联网验证。
- 不要引用不存在、未读取或无法访问的来源。

## 修改代码前

- 修改前必须确认相关入口、调用链、数据结构、配置来源和现有测试方式。
- 如果工作区已有未提交改动，必须避免覆盖无关改动；涉及同一文件时要先理解现有变更。
- 只改和任务直接相关的文件。不要顺手重构无关代码、格式化大文件或修改无关配置。
- 对数据库、认证、支付、权限、删除、迁移、批量更新等高风险改动，必须说明影响范围和兼容策略。

## 修改代码后

- 完成代码或配置修改后，必须运行可行的验证，例如编译、测试、静态检查、SQL 校验、接口冒烟或页面检查。
- 如果验证无法运行，必须说明原因、风险和建议的手动验证步骤。
- 最终回答必须包含：改了什么、验证结果、仍需注意的风险。
- 不要声称“已修复”或“已验证”，除非确实执行过对应验证。

## URL 和外部资料

- 不要编造 URL。URL 必须来自用户提供、项目文件读取或联网搜索结果。
- 对外部文档、软件版本、依赖能力、API 规范等信息，优先使用官方文档或项目仓库。
- 如果用户明确要求不要联网，则只能基于本地文件和已知上下文回答，并说明信息可能不是最新。

## 不确定时

- 直接说明不确定点，不要隐藏在模糊措辞里。
- 给出下一步如何验证，例如要查哪个文件、运行哪个命令、访问哪个官方文档。
- 可以提供保守建议，但必须标明它是建议而非已验证事实。

## 本地部署任务协议

- `DEPLOYMENT.md` 是本仓库本地部署的唯一事实源；部署前必须读取它以及 `.env.example`、`docker-compose.yml` 和 `back/src/main/resources/application.yml`。
- 第一步运行只读诊断：Windows 使用 `powershell -ExecutionPolicy Bypass -File .\scripts\doctor.ps1`，Linux/macOS 使用 `bash ./scripts/doctor.sh`。
- 默认使用 `scripts/deploy.ps1` 或 `scripts/deploy.sh`，不得另写依赖本机绝对路径的临时启动方案。
- 部署脚本不安装系统软件。缺少 JDK 或 Node 时，报告缺失项并等待所有者处理或授权；缺少 Docker 时，仅在本机 MySQL/Redis 诊断通过后使用本机服务流程。
- Docker CLI 可用不代表 daemon 可用；必须通过 `docker info` 验证。daemon 不可用但本机 MySQL/Redis 已启动时，可选择 `-SkipInfrastructure` / `--skip-infrastructure`。
- Docker 不可用时，doctor 使用 `-SkipDocker` / `--skip-docker` 检查 `.env` 指向的本机 MySQL/Redis；首次空库使用 `scripts/init-local-db.ps1` / `scripts/init-local-db.sh`，不要手写字符串管道导入 SQL。
- `-InitDb` / `--init-db` 仅允许用于空数据库。导入失败时保留原数据库并改用新 `DB_NAME` 重试；禁止自动删除 `mysql-data/`、`redis-data/`、Docker volume、已有表或用户数据。
- 端口冲突时先识别占用进程，不得直接终止未知进程。自定义 MySQL/Redis 端口通过根目录 `.env` 统一设置。
- 完成标准：后端 `/api/actuator/health` 返回 `UP`，用户端 `5173` 和管理端 `5174` 返回 HTTP 200，并报告实际命令与日志位置。

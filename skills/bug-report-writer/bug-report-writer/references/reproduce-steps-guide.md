# 复现步骤编写指南

## 1. 核心原则

### 1.1 三可原则

| 原则 | 含义 | 验证方法 |
|------|------|----------|
| **可执行** | 他人能按步骤操作 | 让不熟悉该功能的人按步骤操作 |
| **可复现** | 每次执行都能出现同样问题 | 至少复现2次再提Bug |
| **可验证** | 能明确判断是否修复 | 预期结果可量化/可观察 |

### 1.2 最小化原则

- 只保留**必要的**步骤
- 去除与问题无关的操作
- 不要合并多个步骤

## 2. 步骤编写模板

### 2.1 基础模板

```
1. 打开[APP/页面]，进入[目标页面]
2. [核心操作]
3. [触发操作]
4. 观察[异常现象]
```

### 2.2 带前置条件的模板

```
前置条件：
- 已登录账号：test@example.com
- 购物车中已有2件商品

1. 进入购物车页面
2. 点击商品数量「+」按钮
3. 观察合计金额
```

### 2.3 带特定环境的模板

```
环境要求：
- Android 13，Samsung Galaxy S23
- 网络环境：弱网（2G/Edge）

1. 打开APP，进入首页
2. 下拉刷新
3. 观察页面加载状态
```

## 3. 不同Bug类型的步骤模板

### 3.1 功能缺陷

```
1. 打开APP，进入[目标页面]
2. [执行操作A]
3. [执行操作B]
4. 实际：[异常现象]
   预期：[正确行为]
```

### 3.2 UI/布局缺陷

```
1. 打开APP，进入[目标页面]
2. [如需滚动] 滑动到[位置]
3. 观察[元素]的显示状态

观察到的UI异常：
- [具体描述：位置/大小/颜色/文字]

设备信息：
- 分辨率：[如1080×2400]
- 字体大小：[如超大字体]
- 横竖屏：[竖屏/横屏]
```

### 3.3 性能问题

```
1. 清除APP缓存（确保冷启动）
2. 打开APP，进入[目标页面]
3. 记录操作开始时间 T1 = [时间]
4. 执行[操作]
5. 记录操作完成时间 T2 = [时间]
6. 计算响应时间 = T2 - T1 = [X秒]

性能指标：
- 实际耗时：[X秒]
- 预期耗时：[Y秒以内]
- 超出预期：[Z秒]
```

### 3.4 崩溃/闪退

```
1. 打开APP，进入[目标页面]
2. [执行操作A]
3. [执行操作B]
4. APP崩溃/闪退

崩溃信息：
- 崩溃类型：[ANR / Java Crash / Native Crash]
- 崩溃时间：[时间戳]
- 堆栈信息：[附后]

恢复状态：
- 重新打开APP后[描述状态]
```

### 3.5 间歇性Bug

```
1. 打开APP，进入[目标页面]
2. [执行操作]
3. 问题间歇性出现

复现概率：
- 尝试次数：[如10次]
- 出现次数：[如3次]
- 复现概率：[如30%]

出现条件（如有规律）：
- [如：仅在首次进入页面时出现]
- [如：仅在快速连续点击时出现]

出现时现象：[描述]
未出现时行为：[描述]
```

### 3.6 接口异常

```
1. 发送请求：[HTTP方法] [URL]
2. 请求参数：
   [参数列表]
3. 观察响应

请求信息：
- Method: [GET/POST/PUT/DELETE]
- URL: [/api/v1/orders]
- Headers: {Authorization: Bearer xxx, Content-Type: application/json}
- Body: {key: value}

响应信息：
- Status Code: [500]
- Response Body: {"code": 500, "msg": "内部错误"}
- 响应时间: [3000ms]
```

### 3.7 数据异常

```
1. 进入[页面/功能]
2. [查看数据]

数据异常详情：
- 期望数据：[描述正确数据]
- 实际数据：[描述错误数据]
- 差异点：[列出具体差异]

数据来源：
- 接口：[/api/v1/user/profile]
- 数据库表：[user_table]
- 缓存：[Redis key: user:xxx]
```

## 4. 步骤质量检查

### 4.1 自查清单

- [ ] 步骤是否有编号？
- [ ] 每步是否只包含一个操作？
- [ ] 操作对象是否具体明确（不用"某按钮"）？
- [ ] 是否包含必要的前置条件？
- [ ] 是否标注了必要的环境要求？
- [ ] 他人按步骤能否复现？
- [ ] 偶现Bug是否标注了复现概率？

### 4.2 常见错误

| 错误 | ❌ 差 | ✅ 好 |
|------|-------|-------|
| 步骤合并 | 1. 输入账号密码点击登录 | 1. 输入账号test@example.com 2. 输入密码Test123 3. 点击「登录」 |
| 对象模糊 | 点击那个按钮 | 点击「提交订单」按钮 |
| 缺前置 | 1. 点击购买 | 前置：已登录且有收货地址 1. 点击「立即购买」 |
| 无预期 | 页面不对 | 预期：跳转订单确认页；实际：页面白屏 |

## 5. 高级技巧

### 5.1 条件分支步骤

```
1. 进入购物车页面
2. 如果购物车为空：
   a. 添加任意商品到购物车
3. 选择商品，点击「-」按钮将数量减至0
4. 观察页面变化
```

### 5.2 循环操作步骤

```
1. 进入商品详情页
2. 连续快速点击「收藏」按钮5次
3. 观察收藏状态和请求情况
```

### 5.3 对比验证步骤

```
1. 进入订单列表页
2. 记录订单A的金额（如¥199.00）
3. 进入订单详情页
4. 对比详情页金额与列表页是否一致
```

# ProjectKu 监控体系配置说明

## 📦 已添加的组件

| 组件 | 版本 | 作用 |
|------|------|------|
| Spring Boot Actuator | 与 Spring Boot 一致 | 暴露应用健康状态和指标 |
| Micrometer Prometheus | 最新 | 指标收集，桥接到 Prometheus |
| Prometheus | latest | 时序数据库，存储和查询指标 |
| Grafana | latest | 可视化展示，创建仪表板 |

---

## 🚀 启动步骤

### 1. 确保 Docker 已启动
```bash
# 检查 Docker 是否运行
docker --version
```

### 2. 启动监控服务
```bash
# 在项目根目录执行
docker-compose up -d
```

### 3. 启动 Spring Boot 应用
```bash
cd back
mvn spring-boot:run
```

## 📊 Grafana 仪表板配置

### 第一步：添加 Prometheus 数据源

1. 打开 http://localhost:3000
2. 登录（admin / admin）
3. 点击左侧菜单 **Connections** → **Data sources**
4. 点击 **Add data source**
5. 选择 **Prometheus**
6. 配置：
   - URL: `http://prometheus:9090`
7. 点击 **Save & test**

### 第二步：导入预配置的仪表板

推荐导入这些官方仪表板：

| 仪表板 | ID | 描述 |
|--------|----|------|
| **JVM (Micrometer)** | 4701 | JVM 内存、GC、线程监控 |
| **Spring Boot Statistics** | 10280 | Spring Boot 应用指标 |
| **HikariCP Connection Pool** | 10856 | 数据库连接池监控 |
| **Prometheus Stats** | 3662 | Prometheus 自身状态 |

导入步骤：
1. 点击左侧菜单 **Dashboards** → **New** → **Import**
2. 输入仪表板 ID（如 4701）
3. 点击 **Load**
4. 选择 Prometheus 数据源
5. 点击 **Import**

## 📈 主要监控指标

### 应用指标
- `http.server.requests` - HTTP 请求统计
- `jvm.memory.used` - JVM 内存使用
- `jvm.gc.pause` - GC 停顿时间
- `hikaricp.connections.active` - 活跃数据库连接

---

## 🛑 停止监控服务

```bash
docker-compose down
```

如需保留数据：
```bash
docker-compose stop
```

---

## 🔍 验证监控是否正常工作

### 检查 Prometheus 目标

1. 打开 http://localhost:9090
2. 点击菜单 **Status** → **Targets**
3. 应看到 `spring-boot` 状态为 **UP**

### 检查 Grafana 数据

1. 打开 http://localhost:3000
2. 进入已导入的仪表板
3. 应能看到实时数据图表

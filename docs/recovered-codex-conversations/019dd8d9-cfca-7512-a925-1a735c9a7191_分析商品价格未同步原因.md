## 助手 · 2026-04-29T10:50:27.244Z

原因基本确定：你在管理端改的“售价”是 `products.price`，但商品详情页在商品有 SKU 时展示的是 `product_skus.price`，所以基础售价改了，详情页价格不会跟着变。

我本地接口验证了商品 `1`：接口返回 `price = 7999`，但 `skus[].price = 8999 / 10999 / 12999`。前台详情页会显示 SKU 价格区间，而不是 `7999`。

关键链路：

- 管理端保存时同时提交 `price: form.price` 和 `skus[].price`： [ProductsView.vue](<D:\Java\class\projectKu\web\frontend-admin\src\views\ProductsView.vue:352>)
- 后端更新 `products.price`： [AdminProductController.java](<D:\Java\class\projectKu\web\back\src\main\java\com\web\controller\AdminProductController.java:263>)
- 后端也会用提交的 SKU 列表重写 `product_skus.price`： [AdminProductController.java](<D:\Java\class\projectKu\web\back\src\main\java\com\web\controller\AdminProductController.java:227>)
- 前台详情页只要有 `x.skus`，就用 SKU 价格构建展示数据： [ProductDetailView.vue](<D:\Java\class\projectKu\web\frontend\src\views\ProductDetailView.vue:693>)
- 最终价格文本取选中 SKU 价格，未选中时取 SKU 最小/最大价格： [ProductDetailView.vue](<D:\Java\class\projectKu\web\frontend\src\views\ProductDetailView.vue:371>)

缓存大概率不是主因：管理端更新接口已经加了 `@CacheEvict` 清理 `product_detail` 和 `product_list`。但如果详情页已经打开，它只在 `onMounted` 时加载一次，不会自动感知后台改价，需要刷新页面才会重新请求。

建议统一规则：有 SKU 的商品，要么管理端提示/要求改 SKU 价格，要么保存基础售价时同步 SKU 价格，要么详情页改为未选规格时优先展示 `products.price`。当前代码的行为是“详情价以 SKU 为准”。

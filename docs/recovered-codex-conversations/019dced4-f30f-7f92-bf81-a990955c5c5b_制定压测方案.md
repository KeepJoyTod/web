**执行方案**

1. 准备环境  
- 启动依赖：`docker compose up -d`
- 启动后端：在 `back/` 下执行 `mvn spring-boot:run`
- 确认服务：`http://127.0.0.1:8080/api/`
- 默认账号可用于登录态接口压测：`user@example.com / 123456`

2. 先做 3 类压测脚本  
- 公开读接口：`GET /v1/categories`、`GET /v1/products`、`GET /v1/products/{id}`
- 登录态读接口：`POST /v1/auth/login`、`GET /v1/me`、`GET /v1/orders`
- 写接口链路：`POST /v1/cart/items`、`POST /v1/orders/checkout`、必要时 `POST /v1/payments/{orderId}/pay`

3. 按阶段执行，不要一上来压下单链路  
- 基线压测：`10 -> 30 -> 50 VUs`，每档 `3m`
- 常规负载：`50 -> 100 VUs`，`5m`
- 峰值压测：`100 -> 200 VUs`，`5m`
- 极限压测：继续加压，直到错误率或延迟明显失控
- 写接口单独压，且并发低于读接口，避免测试数据污染太快

4. 推荐场景配比  
- 浏览型混合流量：商品列表 `45%`，商品详情 `35%`，分类 `10%`，登录 `5%`，购物车 `3%`，下单 `2%`
- 如果目标是验证数据库和事务能力，再单独跑“登录 -> 加购 -> 下单”闭环

5. 验收指标  
- `http_req_failed < 1%`
- `checks > 99%`
- `http_req_duration p95 < 800ms`
- `http_req_duration p99 < 1500ms`
- 同时关注 `vus_max`、吞吐量、JVM 日志、MySQL 连接数

当前项目的发展分支错误上传到了main默认分支，要如何把它迁移到develop分支，清空main分支

## 助手 · 2026-04-29T08:29:29.651Z

可以，核心思路是：**先把当前 `main` 的内容保留下来推到 `develop`，确认无误后再处理 `main`**。

推荐步骤如下：

```bash
git fetch origin

# 1. 从当前 main 创建 develop
git switch main
git pull origin main
git switch -c develop

# 2. 推送 develop 到远端
git push -u origin develop
```

然后去 GitHub / GitLab / Gitee 仓库设置里，把默认分支从 `main` 改成 `develop`。如果不改默认分支，很多平台不允许删除或强制清空 `main`。

确认 `develop` 已经正常后，有两种处理 `main` 的方式。

**方案 A：直接删除远端 main**

```bash
git push origin --delete main
```

如果平台提示不能删除默认分支，说明你还没把默认分支改成 `develop`。

**方案 B：保留 main 分支，但清空内容**

```bash
git switch --orphan empty-main
git rm -rf .
git commit --allow-empty -m "chore: clear main branch"
git branch -M main
git push --force-with-lease origin main
```

本地如果还想回到开发分支：

```bash
git switch develop
```

注意：`git push --force-with-lease origin main` 会重写远端 `main` 历史，执行前最好确认团队没人还依赖 `main` 上的提交。最稳妥的顺序是：**创建并推送 `develop` → 修改默认分支 → 保护 `develop` → 删除或清空 `main`**。

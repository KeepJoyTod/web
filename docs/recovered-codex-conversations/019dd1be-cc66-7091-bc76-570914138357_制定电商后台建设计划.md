## 助手 · 2026-04-28T01:42:35.010Z

我建议按“先可用、再完善”的后台管理平台来做，直接贴合你当前项目的 Vue3 + Spring Boot + MyBatis 架构。

**推荐方案**

后台单独做一个管理端前端，例如新增 `frontend-admin/`，不要混在当前用户商城前端里。原因是商城端偏移动购物体验，后台端偏 PC 管理台，两者布局、权限、路由、交互完全不同。后端继续复用当前 `back/`，新增 `/v1/admin/**` 管理接口。

**一期 MVP：后台能管核心业务**

1. 权限基础
   - 用户表增加 `role`、`status` 字段，或新增 `admin_users` 表。
   - JWT 中写入角色信息。
   - 新增管理员拦截器：普通用户只能访问商城接口，管理员才能访问 `/v1/admin/**`。
   - 后台登录页独立于商城登录页。

2. 后台首页
   - 今日订单数、销售额、待发货订单、待处理售后、低库存商品。
   - 最近订单列表。
   - 简单趋势图可以先后置，第一版用数字卡片即可。

3. 商品管理
   - 商品列表：搜索、分类筛选、上下架状态筛选、分页。
   - 新增/编辑商品：名称、分类、描述、标签、价格、原价、库存、状态。
   - SKU 管理：规格、价格、库存。
   - 商品图片管理。
   - 上架、下架、删除。

4. 分类管理
   - 分类列表。
   - 新增、编辑、删除分类。
   - 支持父子分类，但第一版可以只做一级分类。

5. 订单管理
   - 全部订单列表，不再只查当前用户订单。
   - 按订单号、用户、状态、时间筛选。
   - 查看订单详情、商品明细、收货地址、支付信息。
   - 修改状态：待支付、已支付、已发货、已完成、已取消。
   - 发货操作：建议增加物流公司、物流单号字段。

6. 售后管理
   - 售后列表：待处理、已通过、已拒绝、已取消。
   - 售后详情。
   - 审核通过/拒绝。
   - 填写管理员备注。

7. 用户管理
   - 用户列表。
   - 查看用户订单、地址、优惠券、售后记录。
   - 禁用/启用用户。

**二期增强**

- 优惠券管理：创建优惠券、发放给用户、查看使用情况。
- 评价管理：查看评价、隐藏违规评价。
- 库存预警：低库存商品提醒。
- 操作日志：记录管理员登录、商品修改、订单状态变更、售后审核。
- 数据报表：销售额、订单量、热销商品、用户增长。

**后端改造重点**

当前后端已有商品、订单、售后、优惠券、评价等业务接口，但大多是用户端接口，例如订单接口会按当前登录用户过滤。所以后台不要直接复用这些 Controller，而是新增：

- `AdminAuthController`
- `AdminDashboardController`
- `AdminProductController`
- `AdminCategoryController`
- `AdminOrderController`
- `AdminAftersaleController`
- `AdminUserController`
- `AdminCouponController`

数据库建议先补这些字段/表：

- `users.role`
- `users.status`
- `orders.logistics_company`
- `orders.logistics_no`
- `aftersales.admin_remark`
- `admin_operation_logs`

**前端管理端页面**

建议技术栈继续用：

- Vue 3
- Vite
- Pinia
- Vue Router
- Axios

后台页面结构：

```text
frontend-admin/
  src/
    layouts/AdminLayout.vue
    views/
      LoginView.vue
      DashboardView.vue
      ProductsView.vue
      ProductEditView.vue
      CategoriesView.vue
      OrdersView.vue
      OrderDetailView.vue
      AftersalesView.vue
      UsersView.vue
      CouponsView.vue
      LogsView.vue
    stores/
      auth.ts
    lib/
      api.ts
```

**实施顺序**

1. 先做管理员登录和角色鉴权。
2. 再做商品管理，因为它最容易验证后台是否真正可用。
3. 接着做订单管理和发货。
4. 然后做售后审核。
5. 最后补首页统计、用户管理、优惠券和操作日志。

按你当前项目规模，MVP 预计 4 到 6 个开发日可以完成；如果要做得像完整商用后台，包括 RBAC、报表、日志、批量操作和完整测试，大约需要 2 到 3 周。

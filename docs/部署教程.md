# 一键部署教程（自动检查并安装依赖）



在在你自己的电脑本地（PowerShell / CMD）执行

cd 你要项目部署的文件夹在本机的路径例如D:\projectKu\web

cd D:\projectKu\web

git clone https://github.com/KeepJoyTod/web.git

本教程适用于本仓库的前后端分离项目：

- 后端：Spring Boot（默认 `http://localhost:8080/api`）
- 前端：Vue 3 + Vite（默认 `http://localhost:5173`）
- 数据库：MySQL（默认 root/123456，库名 `web`）

仓库已提供一键部署脚本：

- Windows PowerShell：[scripts/deploy.ps1]
- Linux/macOS Bash：[scripts/deploy.sh]

> 说明：脚本会尽量“自动检查并安装缺失依赖”。但在公司/学校电脑上可能因为权限、组策略或网络限制无法自动安装，此时用“只检查不安装”模式即可。

---

## 0. 准备工作

打开终端并进入项目根目录（包含 `frontEnd/`、`back/`、`scripts/` 的那个目录）。

### 数据库方式二选一

**A) 使用 Docker（推荐）**

- 需要安装并启动 Docker Desktop（Windows）或 Docker Engine（Linux）
- 注意：当前仓库的 [docker-compose.yml]里 `image: mysql:8.0` 如果是注释状态，需要取消注释，否则 `docker compose up` 会失败

**B) 使用本机 MySQL**

- 本机已安装并启动 MySQL（并确保 3306 可用）
- 运行脚本时加 `SkipDb`（Windows）/ `--skip-db`（Linux/macOS）

---

## 1) Windows：一键部署（推荐）

在项目根目录打开 PowerShell，首次执行建议这样跑：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\scripts\deploy.ps1 -Mode dev -InitDb
```

执行完成后访问：

- 前端：http://localhost:5173
- 后端：http://localhost:8080/api

### 常用参数

- `-InitDb`：初始化/重置数据库（会执行建表+种子数据脚本）
  - 注意：SQL 脚本包含 `DROP TABLE IF EXISTS`，会覆盖同名表数据；不要对生产库使用
- `-SkipDb`：不启动 Docker MySQL（用于你自己已经有 MySQL 的情况）
- `-NoInstall`：只检查依赖，不进行自动安装（对他人电脑/受限环境更安全）
- `-SkipBuild`：跳过构建（只启动服务）
- `-DryRun`：仅打印将要执行的动作，不实际执行
- `-OpenWindow`：为前后端分别打开一个 PowerShell 窗口运行（默认不弹窗，日志写入 `logs/`）

### 数据库连接参数（可选）

默认值：

- `-DbName web`
- `-DbUser root`
- `-DbPassword 123456`
- `-MysqlContainer projectku-mysql`

如果你修改了容器名或账号密码，按实际情况传参即可。

## 2) Linux/macOS：一键部署

如果没有执行权限，先赋权：

```bash
chmod +x ./scripts/deploy.sh
```

首次运行（dev + 初始化数据库）：

```bash
./scripts/deploy.sh --mode=dev --init-db
```

只检查不安装系统依赖：

```bash
./scripts/deploy.sh --mode=dev --no-install
```

使用本机 MySQL（不启 Docker）：

```bash
./scripts/deploy.sh --mode=dev --skip-db
```

## 3) 生产模式（可选）

生产模式会倾向于：

- 后端：运行打包后的 jar（需要先构建）
- 前端：使用 `npm run preview` 预览构建产物

Windows：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\scripts\deploy.ps1 -Mode prod -InitDb
```

Linux/macOS：

```bash
./scripts/deploy.sh --mode=prod --init-db
```

## 4) 日志与停止

### 日志

默认不弹窗时：

- 前端日志：`logs/frontend.log`
- 后端日志：`logs/backend.log`

### 停止服务

脚本会记录 PID 到 `.pids/`：

- `.pids/frontend.pid`
- `.pids/backend.pid`

Windows 可手动结束进程（任务管理器），或根据 PID 停止：

```powershell
Get-Content .\.pids\backend.pid | ForEach-Object { Stop-Process -Id $_ -Force }
Get-Content .\.pids\frontend.pid | ForEach-Object { Stop-Process -Id $_ -Force }
```

如果用了 Docker MySQL，可停止容器：

```bash
docker compose down
```

## 5) 常见问题排查

### 1) `docker compose up` 失败

优先检查 [docker-compose.yml]是否包含有效的 `image:` 或 `build:`。如果 `image: mysql:8.0` 被注释，需取消注释。

### 2) 3306 / 8080 / 5173 端口被占用

- 3306 常见于本机已安装 MySQL 并占用端口：可改 compose 端口或使用 `SkipDb/--skip-db`
- 8080/5173 被占用：关闭占用进程或修改后端 `server.port` / 前端 Vite 端口

### 3) 自动安装依赖失败（权限/网络/策略）

使用“只检查不安装”模式：

- Windows：`-NoInstall`
- Linux/macOS：`--no-install`

然后按提示自行安装缺失项（JDK 17、Maven、Node.js、Docker 等）。

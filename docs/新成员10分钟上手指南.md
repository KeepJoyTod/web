# 新成员 10 分钟上手指南（Windows）

这份指南按“最快跑起来”的目标设计，默认你在 Windows + PowerShell 环境下操作。

## 0. 你将得到什么

10 分钟内完成以下目标：

- 本地启动 MySQL
- 初始化项目数据库 `web`
- 启动后端服务（`http://localhost:8080/api`）
- 启动前端页面（`http://localhost:5173`）

---

## 1. 前置环境（1 分钟）

请先确认以下工具可用：

- JDK 17
- Maven 3.8+
- Node.js 18+（建议）
- MySQL 8+（或 Docker）
- PowerShell

快速检查命令：

```powershell
java -version
mvn -version
node -v
npm -v
mysql --version
docker -v
```

> 如果你打算用 Docker 启 MySQL，`mysql --version` 不是必需，但后续导入 SQL 时会更方便。

---

## 2. 获取代码并进入项目（30 秒）

```powershell
cd D:\Java\class\projectKu\web
```

---

## 3. 启动 MySQL（2 分钟）

你可以二选一。

### 方式 A：使用 Docker（推荐）

```powershell
docker compose up -d
```

默认配置：

- Host: `localhost`
- Port: `3306`
- User: `root`
- Password: `123456`
- Database: `web`（容器会自动创建）

### 方式 B：使用你本机已有 MySQL

确保本机 MySQL 已启动，并准备好 `root/123456`（或自行改后端配置）。

---

## 4. 初始化数据库（2 分钟）

在项目根目录执行：

```powershell
mysql -uroot -p123456 -e "CREATE DATABASE IF NOT EXISTS web DEFAULT CHARSET utf8mb4;"

$sqls = @(
  "back/sql/schema_v1.sql",
  "back/sql/schema_v2_address.sql",
  "back/sql/schema_v3_payment.sql",
  "back/sql/schema_v4_marketing_aftersales.sql",
  "back/sql/schema_v5_products_tags.sql",
  "back/sql/seed_demo.sql",
  "back/sql/seed_products_categories_1_8.sql"
)

foreach ($f in $sqls) {
  Write-Host "Importing $f ..."
  Get-Content $f -Raw | mysql -uroot -p123456 web
}
```

如果中途报 `Unknown column 'tags'`，说明脚本顺序不对，确保先执行 `schema_v5_products_tags.sql` 再执行 `seed_products_categories_1_8.sql`。

---

## 5. 启动后端（2 分钟）

新开一个 PowerShell 窗口：

```powershell
cd D:\Java\class\projectKu\web\back
mvn spring-boot:run
```

启动后可访问：

- API 根地址：`http://localhost:8080/api`
- Swagger UI：`http://localhost:8080/api/swagger-ui-custom.html`

---

## 6. 启动前端（2 分钟）

再新开一个 PowerShell 窗口：

```powershell
cd D:\Java\class\projectKu\web\frontend
npm install
npm run dev
```

打开浏览器：

- 前端页面：`http://localhost:5173`

说明：前端已配置代理，`/api` 请求会自动转发到 `http://localhost:8080`。

---

## 7. 60 秒验收清单

看到以下结果即代表环境 OK：

- 访问 `http://localhost:5173` 能打开首页
- 访问 `http://localhost:8080/api/swagger-ui-custom.html` 能看到接口文档
- 访问 `http://localhost:8080/api/v1/products` 能返回商品 JSON

---

## 8. 常见问题（快速排错）

- **后端启动失败，提示数据库连接错误**
  - 检查 MySQL 是否运行
  - 检查账号密码是否与 `back/src/main/resources/application.yml` 一致
- **`mvn` 下载依赖失败或本地仓库异常**
  - 尝试：
    ```powershell
    mvn -Dmaven.repo.local="$env:USERPROFILE\.m2\repository" -DskipTests spring-boot:run
    ```
- **前端白屏或接口 404**
  - 确认后端已启动在 `8080`
  - 确认前端是通过 `npm run dev` 启动（走 Vite 代理）

---

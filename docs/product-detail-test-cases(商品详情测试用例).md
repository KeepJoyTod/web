# 商品详情测试用例

商品详情测试用于验证详情接口、详情页渲染、规格选择、数量调整、购物车写入以及未登录拦截等关键行为。

对应自动化脚本：

- `frontend/tests/product-detail.spec.ts`

运行命令：

```powershell
cd d:\Java\class\projectKu\web\frontend
npm.cmd run test:product-detail
```

| 用例编号 | 用例名称 | 前置条件 | 测试步骤 | 预期结果 |
| --- | --- | --- | --- | --- |
| PD-001 | 商品详情 API 返回详情数据 | MySQL、Redis、后端已启动，数据库存在商品数据 | 先请求商品列表获取可用商品 ID，再请求 `GET /api/v1/products/{id}` | 返回 `200`；包含商品 ID、名称、价格、媒体数据、SKU 数据 |
| PD-002 | 商品详情页可渲染核心信息 | 前端可访问，后端已启动，存在可用商品 | 打开 `/products/{id}` | 页面显示商品图、商品标题、价格、规格按钮、评价区域 |
| PD-003 | 无效商品 ID 显示空状态 | 前端可访问 | 打开 `/products/not-a-number` | 页面停留在该详情路由，并显示空状态面板 |
| PD-004 | 选择规格后可购买，数量可增加 | 商品详情页已打开 | 依次选择每组可用规格；点击增加数量 | 加入购物车和立即购买按钮变为可用；数量从 `1` 增加到 `2`，若库存只允许 1 件则按钮保持不可用 |
| PD-005 | 加入购物车写入本地购物车 | 商品详情页已打开，已选择可用规格 | 点击加入购物车 | 显示成功提示；`localStorage.cart:v1` 中存在该商品 |
| PD-006 | 立即购买未登录跳转登录并保留购物车 | 用户未登录，商品详情页已打开，已选择可用规格 | 点击立即购买 | 跳转到登录页并带 `redirect` 参数；本地购物车中保留该商品 |
| PD-007 | 未登录点击收藏跳转登录 | 用户未登录，商品详情页已打开 | 点击收藏按钮 | 跳转到登录页 |
